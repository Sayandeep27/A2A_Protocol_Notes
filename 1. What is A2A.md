# 3ï¸âƒ£ What is an Agent Executor?

The **Agent Executor** is:

> The engine that actually runs the agent

It:

* Takes input
* Uses LLM
* Decides actions
* Calls tools
* Sends messages to other agents
* Returns output

So:

Agent Card â†’ Description
Agent Executor â†’ Execution

* It is basically wrapper of agents to be able to communicate with A2A protocol
* When you build agent using LangGraph or CrewAI and want to connect those agents with others, then you must wrap it inside an A2A Agent Executor.
* You don't need to do this in case of agents made using ADK because both ADK and A2A protocol is created by Google itself.

```python
# Agent Card
research_card = {
    "name": "ResearchAgent",
    "role": "Collects EV market data"

# Agent Executor
class AgentExecutor:
    def __init__(self, card, llm):
        self.card = card
        self.llm = llm

    def run(self, input_text):
        prompt = f"You are {self.card['role']}. Task: {input_text}"
        return self.llm(prompt)

# Execution

output = research_executor.run("Research EV market in India")
```
